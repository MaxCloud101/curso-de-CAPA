# Application

Una Application en ArgoCD es el recurso que le dice a ArgoCD:

> "Toma estos manifiestos desde esta fuente Git/Helm/Kustomize y desplégalos en este clúster y namespace."

Piensa en una Application como una conexión entre Git y Kubernetes.

## Componentes de una Application

Una Application normalmente define:

### 1. Source (origen) → De dónde vienen los manifiestos

Puede ser:

- Repositorio Git
- Helm chart
- Helm chart Git
- Kustomize
- OCI Registry

Ejemplo:

```
source:
  repoURL: https://github.com/user/app.git
  path: manifests/
  targetRevision: main
```

### Destination (destino) → Dónde desplegar

Ejemplo:
```
destination:
  server: https://kubernetes.default.svc
  namespace: production
```

Significa:

"Instala los recursos en el namespace production"

### Project → Restricciones y permisos

Ejemplo:

```
project: default
```

Un Project controla:

- Qué repositorios puede usar
- Qué namespaces puede desplegar
- Qué clusters puede tocar
P- olíticas de seguridad

### Sync Policy → Cómo sincronizar

Puede ser manual:

```
argocd app sync my-app
```
o automática:

```
syncPolicy:
  automated:
    prune: true
    selfHeal: true
```
Automática significa: Si Git cambia → Kubernetes cambia.


## Creacion de una aplicacion simple

### 1. Desde una interfaz visual

Este ejemplo se realizara desde la gui de ArgoCD

### 2. Desde  un archivo yaml

```
example/2.1.1-application.yaml
```

### 3. Desde ArgoCD cli

```
argocd app create simple-app \
  --repo https://github.com/MaxCloud101/curso-de-CAPA-example.git \
  --path examples/1-simple-app \
  --revision HEAD \
  --dest-server https://kubernetes.default.svc \
  --dest-namespace default \
  --project default
```
