# Repositories

Los Repositories son las fuentes desde donde Argo obtiene los manifiestos que va a desplegar. Sin repositorios, Argo CD no tiene nada que sincronizar al clúster. Argo CD soporta principalmente repositorios Git y repositorios Helm.

# Cómo encajan en la arquitectura

```
Git Repository
       |
       v
+---------------+
|   Argo CD     |
|  repo-server  |
+---------------+
       |
       v
+---------------+
| Kubernetes    |
+---------------+
```

Cuando una Application hace sync:

1. Argo CD clona o descarga el repositorio.
2. Genera los manifests (Kustomize, Helm, Jsonnet o YAML puro).
3. Compara Git vs Kubernetes.
4. Aplica los cambios necesarios.

## Tipos de repositorios

### Git Repository

Puede ser:

- GitHub
- GitLab
- Bitbucket
- Azure Repos
- Gitea
- Git interno

Ejemplo:

```
spec:
  source:
    repoURL: https://github.com/company/platform-apps.git
    path: apps/nginx
    targetRevision: main
```

Argo clonará ese repositorio y buscará los manifests dentro de:

```
apps/nginx
```

### Helm Repository

Puede ser:

```
https://charts.bitnami.com/bitnami
https://prometheus-community.github.io/helm-charts
```

Ejemplo:

```
spec:
  source:
    repoURL: https://charts.bitnami.com/bitnami
    chart: nginx
    targetRevision: 18.2.1
```

Aquí Argo no clona Git sino que descarga el chart Helm directamente.


## Repositories configurados vs no configurados

### Repositorio público

Puedes usarlo directamente:

```
repoURL: https://github.com/company/apps.git
```

Siempre que el AppProject lo permita.

No necesitas registrarlo previamente en Argo CD.

### Repositorio privado

Debes registrar credenciales.

Por ejemplo:

```
argocd repo add \
  https://github.com/company/private-repo.git \
  --username myuser \
  --password mytoken
```

O mediante SSH:

```
argocd repo add \
  git@github.com:company/private-repo.git \
  --ssh-private-key-path ~/.ssh/id_rsa
```

Argo almacenará esas credenciales y las utilizará cuando necesite clonar el repositorio.

## Cómo ver los repositorios registrados

argocd repo list

```
TYPE  NAME  REPO
git         https://github.com/company/apps.git
helm        https://charts.bitnami.com/bitnami
```

## Agregando repositorios

### Repositorios Git

HTTPS:
```
argocd repo add \
  https://github.com/company/apps.git
```

Privado:
```
argocd repo add \
  https://github.com/company/apps.git \
  --username myuser \
  --password mytoken
```

SSH:
```
argocd repo add \
  git@github.com:company/apps.git \
  --ssh-private-key-path ~/.ssh/id_rsa
```

### Repositorios Helm

```
argocd repo add \
  https://charts.bitnami.com/bitnami \
  --type helm
```

Helm privado:

```
argocd repo add \
  https://my-harbor.company.com/chartrepo/library \
  --type helm \
  --username admin \
  --password secret
```

## Repositorios en los Projects

Aquí es donde muchas empresas aplican seguridad.

Supongamos este proyecto:

```
spec:
  sourceRepos:
    - https://github.com/company/platform-apps.git
```

Todas las Applications que pertenezcan a ese proyecto:

```
spec:
  project: platform-team
```

solamente podrán usar:

```
https://github.com/company/platform-apps.git
```

Si intentan usar:

```
https://github.com/company/random-repo.git
```

Argo CD rechazará el despliegue.

Los repositories son la fuente real de manifests y los AppProjects controlan cuáles están permitidos.

## Credential Templates

Supongamos que tienes 50 repositorios GitHub privados:

```
https://github.com/company/app1.git
https://github.com/company/app2.git
https://github.com/company/app3.git
...
```

No quieres registrar credenciales 50 veces.

Puedes crear un template:

```
url: https://github.com/company
username: myuser
password: mytoken
```
Argo aplicará esas credenciales automáticamente a todos los repositorios que coincidan con ese prefijo.

## Dónde guarda Argo los repositories

Cuando agregas un repositorio:

```
argocd repo add ...
```

Argo crea internamente un Secret en el namespace argocd:

```
kubectl get secrets -n argocd
```

Verás algo parecido a:

```
repo-123456789
repo-987654321
```

con la etiqueta:

```
argocd.argoproj.io/secret-type: repository
```

También puedes gestionarlos usando Secrets declarativos.
