# Introducción

Argo CD es una herramienta de GitOps para Kubernetes que automatiza el despliegue y mantenimiento de aplicaciones usando repositorios Git como fuente de verdad (source of truth).

## GitOps

GitOps es una metodología para administrar infraestructura y aplicaciones usando Git como fuente única de verdad (Single Source of Truth). En lugar de aplicar cambios manualmente con comandos como kubectl apply, los cambios se definen en archivos declarativos (YAML, Helm, Kustomize, Terraform, etc.) almacenados en un repositorio Git.

Cuando alguien hace un cambio en Git, una herramienta sincroniza automáticamente ese estado con el clúster o entorno.

### Principios principales de GitOps

#### 1. Declarativo

El estado deseado está definido en archivos:
```
replicas: 5
image: nginx:1.27
```
No en comandos manuales:
```
kubectl scale deployment nginx --replicas=5
```

#### 2. Git es la fuente de verdad

El repositorio contiene:

```
repo/
├── apps/
│   └── deployment.yaml
├── helm/
├── kustomize/
└── ingress.yaml
```
Si algo cambia manualmente en Kubernetes, Git sigue siendo el estado correcto.

#### 3. Cambios mediante Pull Requests

Ejemplo:
```
PR:
Cambiar replicas 2 → 5
Actualizar imagen nginx:1.25 → nginx:1.27
```
Ventajas:
```
Auditoría
Historial
Revisiones
Rollback sencillo
```

#### 4. Reconciliación automática

La herramienta verifica continuamente:

Estado deseado (Git):
```
replicas: 3
```
Estado real:
```
replicas: 1
```
Detecta diferencia (drift) y corrige.

### Herramientas populares de GitOps

- Argo CD → Muy usado en Kubernetes
- Flux → CNCF, ligero
- Jenkins X
- Fleet

## Funcionamiento de ArgoCD

1. Guardas manifiestos Kubernetes (YAML, Helm, Kustomize, etc.) en Git.
2. ArgoCD observa continuamente ese repositorio.
3. Cuando detecta cambios, sincroniza el estado del clúster con lo que existe en Git.
4. Si alguien modifica recursos manualmente en Kubernetes (configuration drift), ArgoCD puede revertirlos automáticamente.

Flujo:

```
Git → ArgoCD → Kubernetes
```

## Cómo funciona ArgoCD (flujo básico)

```
Developer
    │
    │ git push
    ▼
Repositorio Git
(manifiestos YAML / Helm / Kustomize)
    │
    │ monitorea cambios
    ▼
ArgoCD
    │
    │ aplica sincronización
    ▼
Cluster Kubernetes
(Pods, Deployments, Services...)
```

ArgoCD compara dos estados:

- Desired State: lo definido en Git
- Live State: lo que realmente existe en Kubernetes

Por ejemplo si son distintos en la cantidad de pods en un deployment:

```
Git: replicas = 3
Cluster: replicas = 1

ArgoCD → detecta diferencia
ArgoCD → corrige a 3
```
