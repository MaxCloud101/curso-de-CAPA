## GitOps

GitOps es una metodologГӯa para administrar infraestructura y aplicaciones usando Git como fuente Гәnica de verdad (Single Source of Truth). En lugar de aplicar cambios manualmente con comandos como kubectl apply, los cambios se definen en archivos declarativos (YAML, Helm, Kustomize, Terraform, etc.) almacenados en un repositorio Git.

Cuando alguien hace un cambio en Git, una herramienta sincroniza automГЎticamente ese estado con el clГәster o entorno.

## Funcionamiento de ArgoCD

1. Guardas manifiestos Kubernetes (YAML, Helm, Kustomize, etc.) en Git.
2. ArgoCD observa continuamente ese repositorio.
3. Cuando detecta cambios, sincroniza el estado del clГәster con lo que existe en Git.
4. Si alguien modifica recursos manualmente en Kubernetes (configuration drift), ArgoCD puede revertirlos automГЎticamente.

Flujo:

```
Git вҶ’ ArgoCD вҶ’ Kubernetes
```

1. API server (argocd-server)

- AutenticaciГіn
- IntegraciГіn SSO (OIDC, LDAP, GitHub)

2. Application Controller 

Es el cerebro de ArgoCD.

Su trabajo: Observar estado deseado

```

```

```

```


### Funciones:
