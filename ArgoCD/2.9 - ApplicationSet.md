# ApplicationSet

ApplicationSet es un recurso de Argo CD que sirve para generar automáticamente uno o muchos objetos Application a partir de una plantilla (template) y una fuente de datos (generator). En lugar de crear manualmente decenas o cientos de Applications, defines una sola vez la lógica y Argo CD las crea por ti.

## ¿Qué problema resuelve?

Imagina que tienes:

- 20 clusters Kubernetes
- 5 aplicaciones
- 3 ambientes (dev, qa, prod)

Sin ApplicationSet podrías terminar administrando cientos de objetos Application.

Con ApplicationSet defines:

> "Por cada cluster registrado en ArgoCD, despliega esta aplicación"

o

> "Por cada directorio encontrado en Git, crea una Application"

y el controlador se encarga del resto.

## Arquitectura

```
ApplicationSet
       │
       ▼
+---------------------+
| ApplicationSet      |
| Controller          |
+---------------------+
       │
       ▼
Genera Applications
       │
       ▼
+-------------+
| Application |
+-------------+
       │
       ▼
Sincroniza recursos Kubernetes
```
Es importante entender que:

- El ApplicationSet Controller solo crea, actualiza o elimina objetos Application.
- El que realmente despliega recursos al cluster sigue siendo Argo CD mediante las Applications generadas.

## Componentes principales

Un ApplicationSet tiene dos partes fundamentales:

### 1. Generator

Define de dónde salen los datos.

Ejemplo:
```
generators:
- list:
    elements:
    - cluster: dev
    - cluster: prod
```
Genera dos conjuntos de parámetros:
```
cluster=dev
cluster=prod
```
### 2. Template

Define cómo construir la Application.
```
template:
  metadata:
    name: '{{cluster}}-guestbook'
```
Resultado:
```
dev-guestbook
prod-guestbook
```

### Ejemplo completo
```
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: guestbook
spec:
  generators:
  - list:
      elements:
      - cluster: dev
        namespace: guestbook-dev

      - cluster: prod
        namespace: guestbook-prod

  template:
    metadata:
      name: '{{cluster}}-guestbook'

    spec:
      project: default

      source:
        repoURL: https://github.com/company/apps.git
        targetRevision: main
        path: guestbook

      destination:
        server: https://kubernetes.default.svc
        namespace: '{{namespace}}'
```
Esto generará automáticamente:
```
Application:
  dev-guestbook

Application:
  prod-guestbook
```

## Generadores más utilizados


### 1. List Generator

Lista estática de elementos.
```
generators:
- list:
    elements:
    - env: dev
    - env: qa
    - env: prod
```
Ideal para laboratorios o pocos ambientes.

### 2. Cluster Generator

Lee automáticamente los clusters registrados en ArgoCD.
```
generators:
- clusters: {}
```
Si mañana agregas un nuevo cluster:
```
argocd cluster add
```
ApplicationSet detectará el nuevo cluster y generará una nueva Application automáticamente.

### 3. Git Directory Generator

Crea una Application por directorio encontrado.

Supongamos este repositorio:
```
apps/
├── app1
├── app2
├── app3
```
Generator:
```
generators:
- git:
    repoURL: https://github.com/company/apps.git
    directories:
    - path: apps/*
```
Resultado:
```
app1
app2
app3
```
Se crean tres Applications automáticamente.

### 4. Git File Generator

Lee archivos YAML o JSON y usa sus valores.
```
clusters/
├── dev.yaml
├── prod.yaml
```
dev.yaml
```
cluster: dev
namespace: guestbook
```
Generator:
```
generators:
- git:
    files:
    - path: clusters/*.yaml
```
Cada archivo genera una Application distinta.

### 5. Matrix Generator

Combina dos generadores.

Ejemplo:
```
3 clusters
x
4 aplicaciones
```
Resultado:
```
12 Applications
```
Muy usado para despliegues multi-cluster

### 6. SCM Provider Generator

Descubre repositorios automáticamente desde:

- GitHub
- GitLab
- Bitbucket
- Azure DevOps

Ejemplo:
```
generators:
- scmProvider:
    github:
      organization: my-org
```
Si aparece un nuevo repositorio en la organización, se crea una Application automáticamente.

## Caso real: Plataforma Kubernetes
```
clusters:
- dev
- qa
- prod

addon:
- ingress-nginx
- cert-manager
- external-secrets
```
Con Matrix Generator:
```
3 clusters × 3 addons
```
ApplicationSet genera:
```
dev-ingress-nginx
qa-ingress-nginx
prod-ingress-nginx

dev-cert-manager
qa-cert-manager
prod-cert-manager

dev-external-secrets
qa-external-secrets
prod-external-secrets
```
Todo desde un único manifiesto.
