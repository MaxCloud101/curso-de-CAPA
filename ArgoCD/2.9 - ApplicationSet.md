# ApplicationSet

ApplicationSet es un recurso de Argo CD que sirve para generar automáticamente uno o muchos objetos Application a partir de una plantilla (template) y una fuente de datos (generator). En lugar de crear manualmente decenas o cientos de Applications, defines una sola vez la lógica y Argo CD las crea por ti.

## ¿Qué problema resuelve?

Imagina que tienes:

- 20 clusters Kubernetes
- 5 aplicaciones
- 3 ambientes (dev, qa, prod)

Sin ApplicationSet podrías terminar administrando cientos de objetos Application.

Con ApplicationSet defines:

> "Por cada cluster registrado en ArgoCD, despliega esta aplicación"

o

> "Por cada directorio encontrado en Git, crea una Application"

y el controlador se encarga del resto.

## Arquitectura

```
ApplicationSet
       │
       ▼
+---------------------+
| ApplicationSet      |
| Controller          |
+---------------------+
       │
       ▼
Genera Applications
       │
       ▼
+-------------+
| Application |
+-------------+
       │
       ▼
Sincroniza recursos Kubernetes
```
Es importante entender que:

- El ApplicationSet Controller solo crea, actualiza o elimina objetos Application.
- El que realmente despliega recursos al cluster sigue siendo Argo CD mediante las Applications generadas.
