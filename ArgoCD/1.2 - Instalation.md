# Instalación

En la pagina oficial vamos a encontrar 4 tipos de instalaciones:
```
https://argo-cd.readthedocs.io/en/stable/operator-manual/installation/
```
- Non-High Availability: Recomendado para pruebas
  - **install.yaml:** Instalación estándar de Argo CD con acceso de administrador de clúster.
  - **namespace-install.yaml:** Instalación de Argo CD que solo requiere privilegios a nivel de namespace (no necesita roles de clúster). Recomendado si necesitas diferentes instancias de ArgoCD para diferentes equipos. Este aun te permite desplegar aplicaciones en el mismo cluster o cluster externo, pero necesitas agregar credenciales (i.e. argocd cluster add <CONTEXT> --in-cluster --namespace <YOUR NAMESPACE>)

- High Availability: Recomendado para produccion
  - **ha/install.yaml:** Igual que install.yaml pero con múltiples réplicas para los componentes compatibles.
  - **ha/namespace-install.yaml:** Igual que namespace-install.yaml pero con múltiples réplicas para los componentes compatibles.

Para nuestras pruebas usaremos la siguiente configuracion:

1. Los CRD de Argo CD no están incluidos en namespace-install.yaml y deben instalarse por separado.

```
kubectl apply --server-side --force-conflicts -k https://github.com/argoproj/argo-cd/manifests/crds\?ref\=stable
```

2. Lanzamos la instalacion sin alta disponibilidad

```
kubectl apply -f https://raw.githubusercontent.com/argoproj/argo-cd/refs/tags/stable/manifests/namespace-install.yaml
```

3. Cambiamos la configuracion de ArgoCD server para servir trafico por el puerto 80

```
kubectl patch deployment argocd-server -n argocd \
  --type='json' \
  -p='[{"op":"add","path":"/spec/template/spec/containers/0/args/-","value":"--insecure"}]'
```

4. Reiniciamos el  deployment de argocd-server
```
kubectl rollout restart deployment argocd-server -n argocd
```

5. Hacemos port fordwar para acceder a argocd desde la web o desde el cli

```
kubectl port-forward svc/argocd-server \
-n argocd \
--address 0.0.0.0 \
8080:80
```

6. El usuario por defecto es admin y para obtener el password lanzamos el siguiente comando:

```
kubectl get secret argocd-initial-admin-secret \
-n argocd \
-o jsonpath="{.data.password}" \
| base64 -d
```
